# Jarvis 브레인스토밍

## 핵심 문제 정의

사람들은 기록의 가치를 알지만 기록하지 않는다. 왜?

- 카테고리 만들어야 함 → 취미 > 영화 > 레전드 > 줄거리, 감상...
- 포맷 맞춰야 함 → 제목, 날짜, 태그, 본문...
- 정보를 직접 찾아서 채워야 함 → 감독, 출연진, 줄거리...
- **결국 "안 적고 말지"가 된다**

회의도 마찬가지:
- 회의록 작성 귀찮음
- 프로젝트 A 회의인데 프로젝트 B 얘기도 나옴 → 어디에 적지?
- 그때그때 나온 아이디어 → 기록 안 하면 증발

모니터링도 마찬가지:
- 비교과포인트 800점 채워야 하는데, 프로그램 열리는 걸 맨날 홈페이지 들어가서 확인해야 함
- 예비창업패키지 공고가 2월 말에 올라온다는데 매일 들어가봐야 함
- 나한테 맞는 공모전 찾으려면 여러 사이트를 수시로 체크해야 함
- **결국 놓치거나, 모니터링 자체가 일이 된다**

## 핵심 비전

> **"레전드라는 영화봄 톰하디 개멋있음"**
> 이 한 줄이면 끝. 나머지는 Jarvis가 한다.

입력의 마찰을 0에 수렴시키는 AI 지식 관리 시스템.

## 핵심 메커니즘

### 1. 토픽 자동 추출 (Topic Extraction)
- 비정형 텍스트에서 주제를 자동 식별
- 하나의 입력에 3개 주제가 있으면 3개로 분리
- 회의 녹음에서 프로젝트 A, B 관련 내용을 각각 추출

### 2. 클러스터 매칭 (Cluster Matching)
- 추출된 토픽을 기존 클러스터(카테고리)와 대조
- 매칭되는 클러스터가 있으면 → 해당 클러스터에 문서 추가
- 없으면 → 새 클러스터 자동 생성
- 클러스터 계층 구조도 AI가 판단 (취미 > 영화 등)

### 3. AI 보강 (Enrichment)
- 웹 검색으로 객관적 정보 자동 삽입 (영화 정보, 줄거리 등)
- 사용자의 짧은 감상을 맥락적으로 구체화
  - "톰하디 개멋있음" → "톰하디의 슈트핏과 담배 피는 장면이 인상적이었던 것 같다"
- 사용자 성향/맥락을 학습해서 자연스러운 확장

### 4. 시맨틱 검색 & 회상 (Semantic Retrieval)
- 정확한 키워드가 아니어도 검색 가능
- "느와르 영화인데 슈트핏이 멋있는 거" → "레전드" 매칭
- 사용자가 직접 쓰지 않은 표현이어도 AI가 보강한 내용에서 매칭
- "언제 이 영화 봤는데, 그때 이런 감상을 남겼다. 이거 맞나?" 형태로 회상 지원

### 5. 실시간 캡처 (Real-time Capture)
- 회의 녹음 → 실시간 트랜스크립션
- 회의 중 나온 아이디어를 프로젝트별로 자동 분류
- 프로젝트 A 회의에서 나온 프로젝트 B 관련 내용 → B 클러스터에 별도 기록

### 6. 버전 관리 (Versioning)
- Git 기반 문서 버전 관리
- 프로젝트 B가 처음 언급된 시점 → 해당 시점으로 초기 커밋
- 지식의 변화 이력 추적 가능

### 7. 웹 모니터링 & 자동 알림 (Web Watch)
- URL + 감시 조건을 설정하면 주기적으로 모니터링
- 조건 충족 시 알림 + 해당 클러스터에 자동 문서화
- **단순 알림이 아니라, 감지 → 정보 수집 → 문서 생성까지 자동**

동작 방식:
1. 사용자가 URL과 지시를 등록
   - "이 페이지에서 비교과프로그램 새로 열리면 알려줘"
   - "예비창업패키지 공고 올라오면 알려줘"
   - "이 사이트에서 나한테 맞을 만한 공모전 올라오면 알려줘"
2. 클라우드에서 주기적 크롤링/변경 감지
3. 조건 매칭 시:
   - 사용자에게 알림 발송
   - 해당 내용을 관련 클러스터에 문서로 자동 추가
   - 공고 상세 정보, 마감일, 지원 조건 등 자동 정리

핵심 차별점:
- 기존: 알림만 옴 → 사용자가 직접 들어가서 확인하고 정리
- Jarvis: 알림 + 정보 추출 + 문서화까지 원스톱
- 사용자 프로필 기반 맞춤 필터링 (내 전공, 관심사, 경력 등으로 "나한테 맞는" 판단)

## 사용 시나리오

### 시나리오 1: 일상 기록
```
입력: "레전드라는 영화봄 톰하디 개멋있음"

Jarvis 동작:
1. 토픽 추출: 영화 감상
2. 클러스터 매칭: "영화" 클러스터 존재? → 없으면 생성
3. 웹 검색: 영화 "레전드" 정보 (2015, 감독, 출연진, 줄거리)
4. AI 보강: 감상평 구체화
5. 문서 생성 → 영화 클러스터에 추가
```

### 시나리오 2: 회의
```
입력: [회의 녹음 트랜스크립트]

Jarvis 동작:
1. 토픽 추출: 프로젝트 A 진행사항, 프로젝트 B 초기 논의, 일정 조율
2. 클러스터 매칭: A는 기존, B는 신규 생성
3. 각 프로젝트별 문서 생성/업데이트
4. B는 신규 클러스터 + 초기 커밋
```

### 시나리오 3: 웹 모니터링
```
설정: URL = "비교과통합관리시스템", 조건 = "새 프로그램 등록"

Jarvis 동작:
1. 매일/매시간 페이지 크롤링
2. 새 비교과프로그램 감지
3. 알림: "OO 프로그램이 열렸어. 50포인트, 마감 3/15"
4. 비교과 클러스터에 문서 자동 추가 (프로그램명, 포인트, 마감일, 신청링크)
```

### 시나리오 4: 맞춤 공모전 추천
```
설정: URL = "공모전 사이트", 조건 = "IT/소프트웨어 관련, 대학생 참가 가능"

Jarvis 동작:
1. 주기적 크롤링
2. 새 공모전 감지 → 사용자 프로필과 매칭
3. 알림: "OO 공모전이 올라왔어. 네 프로필이랑 맞을 것 같아"
4. 공모전 클러스터에 문서 추가 (주제, 상금, 마감일, 지원조건, 참가 전략 제안)
```

### 시나리오 5: 나중에 검색
```
입력: "그때 느와르 영화인데 슈트핏 멋있는 거 뭐였지?"

Jarvis 동작:
1. 시맨틱 검색: 느와르 + 슈트핏 + 멋있는
2. 매칭: 영화/레전드 (AI 보강 내용에 슈트핏 관련 언급)
3. 응답: "2024년 3월 15일에 영화 '레전드'를 보고 이런 감상을 남겼는데, 이거 맞나?"
```

## 차별점

| 기존 도구 (노션 등) | Jarvis |
|---|---|
| 카테고리 직접 생성 | 자동 클러스터링 |
| 포맷 직접 설정 | 자동 문서화 |
| 정보 직접 검색/입력 | 웹 검색 자동 보강 |
| 키워드 기반 검색 | 시맨틱 검색 + 회상 |
| 단일 프로젝트 정리 | 멀티 토픽 자동 분리 |
| 수동 기록 | 음성/텍스트 실시간 캡처 |
| 직접 모니터링 | URL 등록하면 자동 감시 + 문서화 |
| 알림만 제공 | 알림 + 정보 정리 + 클러스터 문서화 |
